Assembly

Hello Assembly

We have been given two files:

Task 1

First we would like to compile hello_assembly.c using the GCC and Clang compiler.

The GCC compiler can be used with the following statement.

gcc -S hello_assembly.c

The Clang compiler can be used with the following statement.

clang -S hello_assembly.c

Task 2

After compilation we would like for each generated assembly file to:

  1. Find the “Hello Assembly Language!” string

  2. Identify the instructions that the compilers insert to conform to the procedure call standard

  3. Identify the function call to libc that prints the string

GCC

The GCC gcc_hello_assembly.s looks like:

  1. The string “Hello Assembly Language!” can be found in line 7.

  2. The instructions for the procedure call standard:

stp x29, x30, [sp, -16]!
mov x29, sp
ldp x29, x30, [sp], 16
ret

  1. The function call is that prints the string:

bl  puts

Clang

The Clang clang_hello_assembly.s looks like:

  1. The string “Hello Assembly Language!” can be found in line 31.

  2. The instructions for the procedure call standard:

stp x29, x30, [sp, -16]!
mov x29, sp
ldp x29, x30, [sp], 16
ret

  1. The function call is that prints the string:

bl  printf

Task 3

Now we would like to test the hello_assembly function with a C++ driver.

To compile and run this code, we can use:

g++ driver_hello_assembly.cpp gcc_hello_assembly.s -o hello_assembly
./hello_assembly

The output is then:

Calling hello_assembly ...
Hello Assembly Language!
... returned from function call!

Assembly Function

Now we want to work with the add_values.s file.

Task 1

As a first step we want to assemble the file. We can do that by calling:

as add_values.s -o add_values.o

Task 2

With add_values.o we are now creating different files:

To get a hexadecimal dump we used:

hexdump add_values.o > hexdump_add_values.hex

To get the section headers we used:

readelf -S add_values.o > sec_headers_add_values.relf

To get the disassembled file we used:

objdump --syms -S -d add_values.o > dis_add_values.dis

Task 3

The size of the .text section can be found in the section headers in line 9.

The size of the text section is 0000000000000020 or 0x20 which translates to 32 bytes. These 32 bytes correspond to 8 assembly instructions (each 4 bytes) that the function add_values performs.

Starting at 0x00, incrementing always by 4 bytes per instruction (performing 8 of those increments), we ultimately finish at the ret instruction located at 0xc1, which marks the end of the .text section. Therefore, from 0x00 to 0xc1 there are exactly 8 instructions covered, which equal a size of 32 bytes.

Task 4

To test the functionality of the add_values function we implemented a C++ driver:

To test our function we now could call:

g++ driver_add_values.cpp add_values.s -o driver_add_values

Where driver_add_values results in:

Calling assembly 'add_value' function ...
l_data_1 / l_value_2 / l_value_o
4 / 7 / 11

Task 5

To get an idea how the contents of the general purpose registers look like, when calling the add_values function we stepped through a function call using the GNU Project Debugger.

To get the correct view in the GNU Project Debugger we call:

gdb ./driver_add_values
lay next

After lay next we pressed enter 3 times to get to the register view in combination with the assembly instructions. To start the debugging process we call:

break add_values
run

After starting the debugging process we then proceeded to skip from instruction to instruction using nexti.
